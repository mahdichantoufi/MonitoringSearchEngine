les requêtes :
Fonctionnement du moteur de recherche :

Le constructeur de requêtes sert à aggréger les résultats des différentes sous requêtes. Il analysera dans un premier temps les sélecteur afin d'éxecuter les requetes avec le sélecteur 'OU' et directement aggréger les résultats avec ceux de la ligne précédente.

Exemple : (R représente une sous-requête, soit une ligne du formulaire)
R1 ET R2 OU R3 SAUF R4 OU R5 ET R6
s'écrit aussi :
R1 ET ( R2 OU R3) SAUF (R4 OU R5) ET R6

Les mots-clé à cocher :

Les mots clés à cocher sont considérés par le moteur de recherche comme étant des sous-requêtes obligatoires.
Ils sont donc des lignes du formulaire avec un sélecteur ET (obligatoire) et une recherche de type expression exacte.

Options :

Vous pouvez ajouter autant de lignes que vous souhaitez dans le formulaire.

- sélecteurs :
'Titre': Mots présents dans le titres. il est conseillé de mettre un mot par sous-requête.
'Tous les mots': Tous ces mots devront apparaître au moins une fois dans les documents retournés.
'Un des mots': si un de ces mots est retrouvé dans le documents, ce dernier est retourné
'Expression exacte': Tous ces mots devront apparaître dans cette ordre exacte.

Les langues présentes au dessus du formulaire servent à signaler la langue originale de la requête afin d'avoir une traduction optimale.
'fr', 'en', 'es', 'it'.

mots-clé Add - Edit - Delete :

Pour ajouter un mot-clé, il suffit de cliquer sur la catégorie à laquelle vous souhaitez l'attribuer puis renseigner les champs.

Pour Supprimer ou modifier un mot-clé, il suffit de cliquer sur la catégorie dans laquelle il se trouve, ensuite cliquer dessus, puis renseigner les nouvelles valeurs.

L'ajout d'un document pdf à travers le backend

Dans le cas où plusieurs documents sont à insérer, vous pourrez mettre tous vos documents PDF accompagnés de leurs fichiers .txt récupérés sur un site de conversion.

Dans le backend, mettre les documents cités précédemment dans le répertoire suivant :
/backend/Files/PDFs_to_insert .
Il suffit ensuite de lancer le script qui les traitera en lançant la commande suivante à la racine du dossier /backend/ :
npm run insertpdfs
